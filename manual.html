🛡️ Moderation & Security

🔒 /mod settings

Here you can choose which channel you will receive most of the server moderation-related logs (for example: warnings, kicks, bans, and more)

1. Log channel

In this modal you can select which channel to receive the logs.

🔒 /mod antiraidsettings

Here you can configure everything related to anti-raid and anti-bot protection.

1. Bot welcome scanner active

This simply serves to activate or deactivate user scanning when someone new enters the server.

2. Make a bot scan in the server

Here you can scan your entire server to find potential bots already present. You can set your own criteria, such as:
1. Whether they have a profile picture
2. Whether they joined Discord within the last certain number of months
3. Whether they don't have a specific role

3. Bot scanner detection config

Here you can configure the requirements related to the scan in section "1." to suit your own criteria:
1. If not having a profile picture would count as a condition for being a bot
2. Minimum account age to avoid being counted as a bot

4. Anti-raid active

This simply activates or deactivates the anti-raid.

5. Anti-raid config

Here you can configure the anti-raid activation criteria:
1. How many Discord invites does someone have to send in less than 1 minute for the anti-raid to be triggered
2. How many images on different channels have to be sent in less than 1 minute for the anti-raid to trigger
3. Here you can choose the punishment: timeout/kick/ban

6. Bot warning and anti-raid log channel

Here you can choose which channel you receive notifications about potential bots and anti-raids.

🔒 /honeypot set

This is used to create a honeypot. The goal of the honeypot is that if someone (a bot, or a brainless human) sends a message in the honeypot channel, a trigger with a punishment is activated.

Activating it will require you to fill out a modal to configure basic honeypot settings, such as:

1. The action
2. The honeypot channel
3. The log channel
4. The honeypot title
5. The honeypot description

1. Punishment

Here you can change the type of punishment: kick/ban

2. Send DM + settings

Here you can configure two things:
1. Enable or disable sending a MD message to the user penalized for speaking in the honeypot channel
2. Here you write the message you want to send

3. Title

Here you can change the title of the honeypot.

4. Description

Here you can change the honeypot description.

5. Color

Here you can set a color for the honeypot.

6. Log Channel

Here you can change the log channel.

🔒 /forbidden words

Here you can write banned words on your server; the automod will delete any messages containing those words.

ADD

A modal window will appear with the following information to fill in:
1. List name
2. trigger words
3. Message that the automod sends to the user when a message is deleted
4. roles that the bot ignores

⚠️: To put multiple letters in trigger words, you have to separate them with commas.
For example: hello, goodbye, welcome

EDIT

Here you can edit:
1. Name
2. Trigger words
3. The message sent
4. Ignored bots
5. Enable or disable partial matches

REMOVE

By entering the number from the list, you delete that list.

🔒 /mod kick

This command is used to kick a user; it removes the user from the server, but they can return.

user

Here you need to put the username.

reason

Here you can add a reason for the kick.

🔒 /mod ban

This command is used to ban a user. Once banned, they cannot rejoin; it is necessary to unban them manually.

user

Here you need to put the username.

reason

Here you can add a reason for the ban.

delate last hour messages

Here you can enable or disable the option to delete the last-minute messages of the banned user.

🔒 /mod mute

This command is used to mute a user for a period of time, preventing them from sending messages on the server.

user

Here you need to put the username.

duration

You have to put a number (1,2,3,4,etc) and a quantity (s,m,h,d)
s = seconds
m = minutes
h = hours
d = days
Example: 1d / 1d12h / 30m

reason

Here you can add a reason for the mute.

🔒 /mod unmute

This command is used to remove the unmute from a user.

user

Here you need to put the username.

🔒 /mod lock

This command locks the channel, so nobody can write; only those with higher permissions can speak.

🔒 /mod unlock

This command unlocks the channel, so everyone can talk in the channel again.

🔒 /mod slowmode

This command is used to put the channel in slow mode, creating a cooldown between messages that a user can send.

time

You have to put a number (1,2,3,4,etc) and a quantity (s,m,h,d)
s = seconds
m = minutes
h = hours
d = days
Use OFF to turn it off
Example: 1d / 1d12h / 30m

🔒 /mod clear

This command is used to delete messages in large quantities.

amount

You have to put a number between 1 and 100

🔒 /mod warn add

This command is used to add a warn to a user.

user

Here you need to put the username.

reason

Here you can add a reason for the warn.

🔒 /mod warn remove

This command is used to remove a warn to a user.

user

Here you need to put the username.

number

This is used to remove a specific warning. Leaving it empty will remove all warnings.

🔒 /mod warn view

This command is used to view user warnings; using it will display two options:

1. user warnings

It is used to display warnings for a single user.

2. all warnings

It is used to display warnings for all users who have warnings.

🔒 /mod stats

This command provides a summary of a moderator's statistics, allowing you to assess their effectiveness.

user

You can enter a moderator's name here to view their stats. If you want to view your own stats, leave this field blank.
